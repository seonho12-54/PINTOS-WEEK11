동기화(Synchronization)

제공된 파일 시스템(file system)은 외부 동기화(external synchronization)를 필요로 합니다. 즉, 호출자는 한 번에 하나의 스레드(thread)만 파일 시스템 코드에서 실행되도록 보장해야 합니다. 제출물은 외부 동기화가 필요 없는 더 세분화된 동기화(synchronization) 전략을 채택해야 합니다. 가능한 한 독립적인 엔티티(entity)에 대한 작업은 서로 독립적이어야 하므로, 서로를 기다릴 필요가 없어야 합니다.

서로 다른 캐시(cache) 블록(block)에 대한 작업은 독립적이어야 합니다. 특히 특정 블록(block)에 I/O가 필요할 때, I/O가 필요 없는 다른 블록(block)에 대한 작업은 그 I/O가 완료될 때까지 기다리지 않고 진행할 수 있어야 합니다.

여러 프로세스(process)가 하나의 파일(file)에 동시에 접근할 수 있어야 합니다. 하나의 파일(file)에 대한 여러 읽기(read)는 서로를 기다리지 않고 완료될 수 있어야 합니다. 파일(file)에 쓰기(write)해도 파일(file)이 확장되지 않는 경우, 여러 프로세스(process)가 하나의 파일(file)에 동시에 쓸 수도 있어야 합니다. 한 프로세스(process)가 파일(file)을 읽는 동안 다른 프로세스(process)가 그 파일(file)에 쓰고 있다면, 읽기는 쓰기(write)가 전혀 완료되지 않았거나, 모두 완료되었거나, 일부만 완료된 상태를 보여도 됩니다. (하지만 write 시스템 콜(system call)이 호출자에게 반환된 뒤에는, 이후의 모든 읽기 주체가 변경 사항을 보아야 합니다.) 마찬가지로 두 프로세스(process)가 파일(file)의 같은 부분에 동시에 쓸 때, 그 데이터는 서로 뒤섞일 수 있습니다.

반면, 파일(file)을 확장하고 새 영역에 데이터를 쓰는 작업은 원자적(atomic)이어야 합니다. 프로세스(process) A와 B가 모두 어떤 파일(file)을 열어 두었고 둘 다 파일 끝(end-of-file)에 위치해 있다고 가정해 봅시다. A가 읽고 B가 동시에 파일(file)에 쓴다면, A는 B가 쓰는 내용의 전체, 일부 또는 아무것도 읽지 않을 수 있습니다. 그러나 A는 B가 쓰는 데이터 이외의 데이터를 읽어서는 안 됩니다. 예를 들어 B가 쓰는 데이터가 모두 0이 아닌 바이트라면, A는 어떤 0도 보아서는 안 됩니다.

서로 다른 디렉터리(directory)에 대한 작업은 동시에 수행되어야 합니다. 같은 디렉터리(directory)에 대한 작업은 서로를 기다려도 됩니다.

여러 스레드(thread)가 공유하는 데이터만 동기화(synchronization)하면 된다는 점을 기억하십시오. 기본 파일 시스템(file system)에서는 struct file 및 struct dir이 단일 스레드(thread)에서만 접근됩니다.
