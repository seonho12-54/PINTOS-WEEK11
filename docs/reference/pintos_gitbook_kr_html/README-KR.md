# Pintos KAIST 한국어 오프라인 문서 사용법

이 폴더는 정적 HTML 문서입니다. 별도 설치나 빌드 없이 바로 볼 수 있습니다.

## 1. 공유할 때

가장 중요한 점은 `index.html` 파일 하나만 보내면 안 되고, **폴더 전체를 그대로** 보내야 한다는 것입니다.

보내야 하는 것:

- `index.html`
- `introduction/`
- `project1/`
- `project2/`
- `project3/`
- `project4/`
- `appendix/`
- `gitbook/`
- 기타 폴더 안의 CSS, JS, 검색 인덱스 파일

가장 쉬운 방법은 이 폴더 전체를 ZIP으로 압축해서 전달하는 것입니다.

## 2. 받는 사람이 보는 방법

### 방법 A. 그냥 더블클릭해서 보기

1. ZIP을 받았다면 먼저 압축을 풉니다.
2. 압축을 푼 폴더 안으로 들어갑니다.
3. `index.html`을 더블클릭합니다.

이 방식으로도 좌측 목차 클릭, 이전/다음 이동, 검색이 동작하도록 만들어져 있습니다.

### 방법 B. 로컬 서버로 보기

더 안정적으로 보려면 간단한 로컬 서버로 열어도 됩니다.

#### macOS / Linux

터미널에서:

```bash
cd pintos-kaist-ko
python3 -m http.server 8765
```

브라우저에서 다음 주소를 엽니다.

```text
http://127.0.0.1:8765/
```

#### Windows

PowerShell 또는 CMD에서:

```bash
cd pintos-kaist-ko
python -m http.server 8765
```

브라우저에서 다음 주소를 엽니다.

```text
http://127.0.0.1:8765/
```

## 3. 상대방에게 같이 보내면 좋은 안내

아래 문구를 그대로 전달해도 됩니다.

```text
압축을 푼 뒤 폴더 안의 index.html을 열면 바로 볼 수 있습니다.
만약 브라우저에서 더 안정적으로 보고 싶다면,
터미널에서 해당 폴더로 이동한 뒤
python3 -m http.server 8765
를 실행하고 http://127.0.0.1:8765/ 로 접속하면 됩니다.

중요: index.html만 따로 옮기지 말고 폴더 전체를 그대로 유지해야 합니다.
```

## 4. 주의사항

- 폴더 구조를 바꾸면 CSS, 목차, 검색, 내부 링크가 깨질 수 있습니다.
- `gitbook/` 폴더를 빼면 검색과 화면 구성이 동작하지 않습니다.
- 가장 안전한 전달 방식은 **폴더 전체 ZIP 압축본 공유**입니다.
