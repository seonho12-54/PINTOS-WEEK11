스레드(Threads)

스레드(Threads)

struct thread

스레드(thread)를 위한 Pintos의 주요 자료 구조는 struct thread이며, threads/thread.h에 선언되어 있습니다.

struct thread;

스레드(thread) 또는 사용자 프로세스(process)를 나타냅니다. 프로젝트에서 여러분은 자신만의 멤버를 struct thread에 추가해야 합니다. 기존 멤버의 정의를 변경하거나 삭제할 수도 있습니다. 모든 struct thread는 자신에게 할당된 메모리 페이지의 시작 부분을 차지합니다. 페이지의 나머지는 스레드(thread)의 스택(stack)으로 사용되며, 스택은 페이지 끝에서 아래쪽으로 자랍니다. 모습은 다음과 같습니다.

                      4 kB +---------------------------------+
                           |         kernel stack            |
                           |               |                 |
                           |               |                 |
                           |               V                 |
                           |        grows downward           |
                           |                                 |
                           |                                 |
                           |                                 |
                           |                                 |
                           |                                 |
                           |                                 |
                           |                                 |
                           |                                 |
    sizeof (struct thread) +---------------------------------+
                           |             magic               |
                           |          intr_frame             |
                           |               :                 |
                           |               :                 |
                           |             status              |
                           |              tid                |
                      0 kB +---------------------------------+

이 구조에는 두 가지 결과가 따릅니다. 첫째, struct thread가 너무 커지도록 두면 안 됩니다. 너무 커지면 커널(kernel) 스택(stack)을 위한 공간이 부족해집니다. 기본 struct thread의 크기는 몇 바이트에 불과합니다. 아마 1 kB보다 훨씬 작게 유지되어야 합니다. 둘째, 커널(kernel) 스택(stack)이 너무 커지도록 두면 안 됩니다. 스택(stack)이 오버플로하면 스레드(thread) 상태를 손상시킵니다. 따라서 커널(kernel) 함수는 큰 구조체나 배열을 non-static 지역 변수로 할당하면 안 됩니다. 대신 malloc() 또는 palloc_get_page()를 사용한 동적 할당을 사용하십시오 (메모리 할당(Memory Allocation) 참조).

tid_t tid;

스레드(thread)의 스레드(thread) 식별자, 즉 tid입니다. 모든 스레드(thread)는 커널(kernel)의 전체 수명 동안 고유한 tid를 가져야 합니다. 기본적으로 tid_t는 typedef로 int에 대해 정의되어 있으며, 새 스레드(thread)는 초기 프로세스(process)의 1에서 시작해 숫자가 하나씩 증가하는 tid를 받습니다. 원한다면 타입과 번호 매기기 방식을 바꿀 수 있습니다.

enum thread_status status;

스레드(thread)의 상태이며, 다음 중 하나입니다.

  • THREAD_RUNNING

    스레드(thread)가 실행 중입니다. 어느 한 시점에는 정확히 하나의 스레드(thread)만 실행됩니다. thread_current()는 실행 중인 스레드(thread)를 반환합니다.

  • THREAD_READY

    스레드(thread)가 실행할 준비는 되었지만 지금 실행 중은 아닙니다. 다음에 스케줄러(scheduler)가 호출될 때 이 스레드(thread)가 실행 대상으로 선택될 수 있습니다. 준비된 스레드(thread)들은 ready_list라는 이중 연결 리스트에 보관됩니다.

  • THREAD_BLOCKED

    스레드(thread)가 어떤 일을 기다리고 있습니다. 예를 들어 락(lock)을 사용할 수 있게 되거나 인터럽트(interrupt)가 호출되기를 기다립니다. 이 스레드(thread)는 THREAD_READY 상태로 전환될 때까지 다시 스케줄링되지 않으며, 이 전환은 thread_unblock() 호출로 이루어집니다. 이는 스레드(thread)를 자동으로 블록하고 언블록하는 Pintos 동기화(synchronization) 프리미티브 중 하나를 사용해 간접적으로 수행하는 것이 가장 편리합니다 (동기화(Synchronization) 참조). 블록된 스레드(thread)가 무엇을 기다리는지 a priori로 알 방법은 없지만, 백트레이스(backtrace)가 도움이 될 수 있습니다 (백트레이스(Backtraces) 참조).

  • THREAD_DYING

    다음 스레드(thread)로 전환한 뒤 이 스레드(thread)는 스케줄러(scheduler)에 의해 제거됩니다.

char name[16];

스레드(thread)의 이름 문자열, 또는 적어도 그 앞부분 몇 글자입니다.

struct intr_frame tf;

레지스터와 스택(stack) 포인터를 포함해 컨텍스트 스위치(context switch)에 필요한 정보를 저장합니다.

int priority;

스레드(thread) 우선순위(priority)이며, 범위는 PRI_MIN(0)부터 PRI_MAX(63)까지입니다. 숫자가 낮을수록 낮은 우선순위(priority)에 해당하므로 우선순위(priority) 0이 가장 낮고 우선순위(priority) 63이 가장 높습니다. 제공되는 Pintos는 스레드(thread) 우선순위(priority)를 무시하지만, 프로젝트 1에서 우선순위 스케줄링(priority scheduling)을 구현하게 됩니다 (우선순위 스케줄링(Priority Scheduling) 참조).

struct list_elem elem;

스레드(thread)를 이중 연결 리스트에 넣는 데 사용하는 "리스트 원소(list element)"입니다. 그 리스트는 ready_list(실행 준비가 된 스레드(thread)들의 리스트)이거나 sema_down()에서 세마포어(semaphore)를 기다리는 스레드(thread)들의 리스트입니다. 세마포어(semaphore)를 기다리는 스레드(thread)는 준비 상태가 아니고, 그 반대도 마찬가지이므로 이 멤버는 두 역할을 겸할 수 있습니다.

uint64_t *pml4;

프로젝트 2 이후에만 존재합니다. 페이지 테이블(Page Tables)을 참조하십시오.

unsigned magic

항상 THREAD_MAGIC으로 설정됩니다. 이는 threads/thread.c에 정의된 임의의 숫자일 뿐이며 스택(stack) 오버플로를 감지하는 데 사용됩니다. thread_current()는 실행 중인 스레드(thread)의 magic 멤버가 있는 struct thread가 THREAD_MAGIC으로 설정되어 있는지 검사합니다. 스택(stack) 오버플로는 이 값을 바꾸는 경향이 있어 assertion을 트리거합니다. 가장 큰 효과를 얻으려면 struct thread에 멤버를 추가할 때 magic을 끝에 두십시오.

스레드(thread) 함수

threads/thread.c는 스레드(thread) 지원을 위한 여러 public 함수를 구현합니다. 가장 유용한 함수들을 살펴보겠습니다.

void thread_init (void);

스레드(thread) 시스템을 초기화하기 위해 main()이 호출합니다. 주된 목적은 Pintos의 초기 스레드(thread)를 위한 struct thread를 만드는 것입니다. Pintos 로더(loader)가 초기 스레드(thread)의 스택(stack)을 다른 Pintos 스레드(thread)와 같은 위치인 페이지의 맨 위에 두기 때문에 이것이 가능합니다.

thread_init()이 실행되기 전에는 thread_current()가 실패합니다. 실행 중인 스레드(thread)의 magic 값이 올바르지 않기 때문입니다. 많은 함수가 thread_current()를 직접 또는 간접적으로 호출하며, 여기에는 락(lock)을 잠그는 lock_acquire()도 포함되므로, thread_init()은 Pintos 초기화 초기에 호출됩니다.

void thread_start (void);

스케줄러(scheduler)를 시작하기 위해 main()이 호출합니다. idle 스레드(thread), 즉 다른 어떤 스레드(thread)도 준비되어 있지 않을 때 스케줄되는 스레드(thread)를 만듭니다. 그런 다음 인터럽트(interrupt)를 활성화합니다. 이는 부수 효과로 스케줄러(scheduler)를 활성화하는데, 스케줄러(scheduler)는 intr_yield_on_return()을 사용해 타이머(timer) 인터럽트(interrupt)에서 반환될 때 실행되기 때문입니다.

void thread_tick (void);

각 타이머(timer) tick마다 타이머(timer) 인터럽트(interrupt)가 호출합니다. 스레드(thread) 통계를 추적하고 time slice가 만료되면 스케줄러(scheduler)를 트리거합니다.

void thread_print_stats (void);

Pintos 종료 중에 스레드(thread) 통계를 출력하기 위해 호출됩니다.

tid_t thread_create (const char *name, int priority, thread func *func, void *aux);

지정된 우선순위(priority)를 가진 name이라는 새 스레드(thread)를 만들고 시작하며, 새 스레드(thread)의 tid를 반환합니다. 스레드(thread)는 func를 실행하고 aux를 함수의 단일 인자로 전달합니다.

thread_create()는 스레드(thread)의 struct thread와 스택(stack)을 위한 페이지를 할당하고 그 멤버들을 초기화한 다음, 이 스레드(thread)를 위해 가짜 스택(stack) 프레임 집합을 설정합니다. 스레드(thread)는 blocked 상태로 초기화된 뒤 반환 직전에 unblocked 상태가 되며, 이를 통해 새 스레드(thread)가 스케줄될 수 있습니다.

void thread_func (void *aux);

thread_create()에 전달되는 함수의 타입입니다. 해당 aux 인자는 그 함수의 인자로 그대로 전달됩니다.

void thread_block (void);

실행 중인 스레드(thread)를 running 상태에서 blocked 상태로 전환합니다. 이 스레드(thread)에 대해 thread_unblock()이 호출될 때까지 다시 실행되지 않으므로, 그 일이 일어나도록 할 방법을 미리 마련해 두어야 합니다. thread_block()은 매우 저수준이므로 대신 동기화(synchronization) 프리미티브 중 하나를 사용하는 편이 좋습니다 (동기화(Synchronization) 참조).

void thread_unblock (struct thread *thread);

blocked 상태여야 하는 스레드(thread)를 ready 상태로 전환하여 다시 실행될 수 있게 합니다. 이는 스레드(thread)가 기다리던 이벤트가 발생했을 때 호출됩니다. 예를 들어 스레드(thread)가 기다리던 락(lock)을 사용할 수 있게 되었을 때 호출됩니다.

struct thread *thread_current (void);

실행 중인 스레드(thread)를 반환합니다.

tid_t thread_tid (void);

실행 중인 스레드(thread)의 스레드(thread) id를 반환합니다. thread_current ()->tid와 같습니다.

const char *thread_name (void);

실행 중인 스레드(thread)의 이름을 반환합니다. thread_current ()->name과 같습니다.

void thread_exit (void) NO_RETURN;

현재 스레드(thread)를 종료하게 합니다. 절대 반환하지 않습니다.

void thread_yield (void);

CPU를 스케줄러(scheduler)에 양보하고, 스케줄러(scheduler)는 실행할 새 스레드(thread)를 선택합니다. 새 스레드(thread)는 현재 스레드(thread)일 수도 있으므로, 이 함수가 특정 시간 동안 현재 스레드(thread)의 실행을 막아 준다고 의존해서는 안 됩니다.

int thread_get_priority (void);
void thread_set_priority (int new_priority);

스레드(thread) 우선순위(priority)를 설정하고 가져오기 위한 stub입니다. 우선순위 스케줄링(Priority Scheduling)을 참조하십시오.

int thread_get_nice (void);
void thread_set_nice (int new_nice);
int thread_get_recent_cpu (void);
int thread_get_load_avg (void);

고급 스케줄러(advanced scheduler)를 위한 stub들입니다.
