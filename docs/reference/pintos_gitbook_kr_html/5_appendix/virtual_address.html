가상 주소(Virtual Addresses)

64비트 가상 주소(virtual address)는 다음과 같이 구성됩니다.

63          48 47            39 38            30 29            21 20         12 11         0
+-------------+----------------+----------------+----------------+-------------+------------+
| Sign Extend |    Page-Map    | Page-Directory | Page-directory |  Page-Table |  Physical  |
|             | Level-4 Offset |    Pointer     |     Offset     |   Offset    |   Offset   |
+-------------+----------------+----------------+----------------+-------------+------------+
              |                |                |                |             |            |
              +------- 9 ------+------- 9 ------+------- 9 ------+----- 9 -----+---- 12 ----+
                                          Virtual Address

헤더 include/threads/vaddr.h와 include/threads/mmu.h는 가상 주소(virtual address)를 다루기 위한 다음 함수와 매크로를 정의합니다.

#define PGSHIFT { /* Omit details */ }
#define PGBITS { /* Omit details */ }

각각 가상 주소(virtual address)의 오프셋(offset) 부분에 대한 비트 인덱스(0)와 비트 수(12)입니다.

#define PGMASK { /* Omit details */ }

페이지 오프셋(page offset)의 비트는 1로, 나머지는 0으로 설정된 비트 마스크(bit mask)입니다 (0xfff).

#define PGSIZE { /* Omit details */ }

바이트 단위의 페이지 크기(page size)입니다(4,096).

#define pg_ofs(va) { /* Omit details */ }

가상 주소 va에서 페이지 오프셋(page offset)을 추출해 반환합니다.

#define pg_no(va) { /* Omit details */ }

가상 주소 va에서 페이지 번호(page number)를 추출해 반환합니다.

#define pg_round_down(va) { /* Omit details */ }

va가 가리키는 가상 페이지(virtual page)의 시작 주소, 즉 페이지 오프셋이 0으로 설정된 va를 반환합니다.

#define pg_round_up(va) { /* Omit details */ }

va를 가장 가까운 페이지 경계(page boundary)로 올림한 값을 반환합니다.

Pintos의 가상 메모리(virtual memory)는 사용자 가상 메모리(user virtual memory)와 커널 가상 메모리(kernel virtual memory)의 두 영역으로 나뉩니다 (가상 메모리 레이아웃(Virtual Memory Layout) 참조).

둘 사이의 경계는 KERN_BASE입니다.

#define KERN_BASE { /* Omit details */ }

커널 가상 메모리(kernel virtual memory)의 기준 주소(base address)입니다. 기본값은 0x8004000000입니다. 사용자 가상 메모리(user virtual memory)는 가상 주소 0부터 KERN_BASE까지의 범위를 갖습니다. 커널 가상 메모리는 가상 주소 공간(virtual address space)의 나머지를 차지합니다.

#define is_user_vaddr(vaddr) { /* Omit details */ }
#define is_kernel_vaddr(vaddr) { /* Omit details */ }

va가 각각 사용자 또는 커널 가상 주소(kernel virtual address)이면 true를, 아니면 false를 반환합니다.

x86-64는 물리 주소(physical address)가 주어졌을 때 메모리에 직접 접근하는 방법을 제공하지 않습니다. 운영체제 커널(operating system kernel)에서는 이런 기능이 자주 필요하므로, Pintos는 커널 가상 메모리(kernel virtual memory)를 물리 메모리(physical memory)에 일대일(one-to-one)로 매핑하여 우회합니다. 즉, 가상 주소 > KERN_BASE는 물리 주소 0에 접근하고, 가상 주소 KERN_BASE + 0x1234는 물리 주소 0x1234에 접근하며, 머신의 물리 메모리 크기까지 이런 식으로 계속됩니다. 따라서 물리 주소에 KERN_BASE를 더하면 그 주소에 접근하는 커널 가상 주소를 얻습니다. 반대로 커널 가상 주소에서 KERN_BASE를 빼면 대응하는 물리 주소를 얻습니다.

헤더 include/threads/vaddr.h는 이러한 변환을 수행하는 함수 쌍을 제공합니다.

#define ptov(paddr) { /* Omit details */ }

물리 주소 pa에 대응하는 커널 가상 주소(kernel virtual address)를 반환합니다. pa는 0과 물리 메모리의 바이트 수 사이에 있어야 합니다.

#define vtop(vaddr) { /* Omit details */ }

va에 대응하는 물리 주소(physical address)를 반환합니다. va는 커널 가상 주소(kernel virtual address)여야 합니다.

헤더 include/threads/mmu.h는 페이지 테이블(page table)에 대한 연산을 제공합니다.

#define is_user_pte(pte) { /* Omit details */ }
#define is_kern_pte(pte) { /* Omit details */ }

페이지 테이블 엔트리(page table entry, PTE)가 각각 사용자 또는 커널 소유인지 질의합니다.

#define is_writable(pte) { /* Omit details */ }

페이지 테이블 엔트리(page table entry, PTE)가 가리키는 가상 주소(virtual address)가 쓰기 가능(writable)한지 질의합니다.

typedef bool pte_for_each_func (uint64_t *pte, void *va, void *aux);
bool pml4_for_each (uint64_t *pml4, pte_for_each_func *func, void *aux);

PML4 아래의 각 유효한 엔트리(entry)에 대해 보조 값 AUX와 함께 FUNC를 적용합니다. VA는 해당 엔트리의 가상 주소(virtual address)를 나타냅니다. pte_for_each_func가 false를 반환하면 순회를 중단하고 false를 반환합니다.

아래는 func 예시이며, pml4_for_each에 전달할 수 있습니다.

static bool
stat_page (uint64_t *pte, void *va,  void *aux) {
        if (is_user_vaddr (va))
                printf ("user page: %llx\n", va);
        if (is_writable (va))
                printf ("writable page: %llx\n", va);
        return true;
}
