해시 테이블(Hash Table)

해시 테이블(Hash Table)

Pintos는 lib/kernel/hash.c에 해시 테이블(hash table) 자료구조를 제공합니다. 이를 사용하려면 헤더 파일 'lib/kernel/hash.h'를 #include <hash.h>로 포함해야 합니다. Pintos가 제공하는 어떤 코드도 해시 테이블을 사용하지 않으므로, 필요에 따라 그대로 사용하거나, 목적에 맞게 구현을 수정하거나, 무시해도 됩니다.

가상 메모리(virtual memory) 프로젝트의 대부분 구현은 해시 테이블을 사용해 페이지를 프레임으로 변환합니다. 그 밖에도 해시 테이블을 유용하게 쓸 곳이 있을 수 있습니다.

데이터 타입(Data Types)

해시 테이블은 struct hash로 표현됩니다.

struct hash;

전체 해시 테이블을 나타냅니다. struct hash의 실제 멤버는 "불투명(opaque)"합니다. 즉, 해시 테이블을 사용하는 코드는 struct hash 멤버에 직접 접근해서는 안 되며, 그럴 필요도 없어야 합니다. 대신 해시 테이블 함수와 매크로를 사용하십시오.

해시 테이블은 struct hash_elem 타입의 원소에 대해 동작합니다.

struct hash_elem;

해시 테이블에 넣으려는 구조체 안에 struct hash_elem 멤버를 포함하십시오. struct hash와 마찬가지로 struct hash_elem도 불투명합니다. 해시 테이블 원소를 다루는 모든 함수는 실제 해시 테이블 원소 타입의 포인터가 아니라 struct hash_elem 포인터를 실제로 인자로 받고 반환합니다.

해시 테이블의 실제 원소에서 struct hash_elem을 얻거나, 그 반대 방향으로 변환해야 하는 일이 자주 있습니다. 해시 테이블의 실제 원소가 주어졌다면 & 연산자로 그 원소의 struct hash_elem에 대한 포인터를 얻을 수 있습니다. 반대 방향으로 변환할 때는 hash_entry() 매크로를 사용합니다.

#define hash_entry (elem, type, member) { /* Omit details */ }

struct hash_elem 포인터인 elem이 포함되어 있는 구조체에 대한 포인터를 반환합니다. elem이 들어 있는 구조체 이름인 type과, elem이 가리키는 type 내부 멤버 이름인 member를 제공해야 합니다.

예를 들어 h가 struct hash_elem * 변수이고, struct thread 멤버 (struct hash_elem 타입)인 h_elem을 가리킨다고 합시다. 그러면 hash_entry (h, struct thread, h_elem)는 struct thread의 주소, 즉 h가 내부에서 가리키는 구조체의 주소를 산출합니다.

각 해시 테이블 원소에는 키(key)가 들어 있어야 합니다. 키는 원소를 식별하고 구분하는 데이터이며, 해시 테이블 안의 원소들 사이에서 고유해야 합니다. (원소에는 고유할 필요가 없는 비키(non-key) 데이터도 들어 있을 수 있습니다.) 어떤 원소가 해시 테이블 안에 있는 동안에는 그 원소의 키 데이터를 변경해서는 안 됩니다. 필요하다면 원소를 해시 테이블에서 제거한 뒤 키를 수정하고, 다시 삽입하십시오.

각 해시 테이블에 대해 키에 작용하는 두 함수를 작성해야 합니다. 하나는 해시 함수(hash function)이고 다른 하나는 비교 함수(comparison function)입니다. 이 함수들은 다음 프로토타입과 일치해야 합니다.

typedef unsigned hash_hash_func (const struct hash_elem *e, void *aux);

원소 데이터의 해시값을 unsigned int 범위 안의 값으로 반환합니다. 원소의 해시는 그 원소 키의 의사 난수 함수여야 합니다. 원소 안의 비키 데이터나 키가 아닌 비상수 데이터에 의존해서는 안 됩니다. Pintos는 해시 함수를 작성할 때 적절한 기반으로 사용할 수 있도록 다음 함수들을 제공합니다.

unsigned hash_bytes (const void *buf, size t *size)

buf에서 시작하는 size 바이트의 해시를 반환합니다. 구현은 32비트 워드용 범용 Fowler-Noll-Vo 해시 (http://en.wikipedia.org/wiki/Fowler_Noll_Vo_hash)입니다.

unsigned hash_string (const char *s)

널 종료 문자열 s의 해시를 반환합니다.

unsigned hash_int (int i)

정수 i의 해시를 반환합니다.

키가 적절한 타입의 단일 데이터라면 해시 함수가 이 함수들 중 하나의 출력을 그대로 반환해도 좋습니다. 여러 데이터 조각으로 이루어진 키라면, 예를 들어 '^'(exclusive or) 연산자를 사용해 이 함수들을 여러 번 호출한 결과를 조합할 수 있습니다. 마지막으로, 이 함수들을 완전히 무시하고 해시 함수를 처음부터 직접 작성해도 됩니다. 다만 목표는 해시 함수를 설계하는 것이 아니라 운영체제 커널(kernel)을 만드는 것임을 기억하십시오. aux에 대한 설명은 [해시 보조 데이터(Hash Auxiliary Data)] 절을 참조하십시오.

bool hash_less_func (const struct hash_elem *a, const struct hash_elem *b, void *aux)

원소 a와 b에 저장된 키를 비교합니다. a가 b보다 작으면 true를 반환하고, a가 b보다 크거나 같으면 false를 반환합니다. 두 원소가 같다고 비교된다면 두 원소의 해시값도 같아야 합니다.

aux에 대한 설명은 [해시 보조 데이터(Hash Auxiliary Data)] 절을 참조하십시오. 해시 함수와 비교 함수 예제는 [해시 테이블 예제(Hash Table Example)] 절을 참조하십시오. 몇몇 함수는 세 번째 종류의 함수에 대한 포인터를 인수로 받습니다.

void hash_action_func (struct hash_elem *element, void *aux)

호출자가 선택한 어떤 동작을 element에 수행합니다. aux에 대한 설명은 [해시 보조 데이터(Hash Auxiliary Data)] 절을 참조하십시오.

기본 함수(Basic Functions)

이 함수들은 해시 테이블을 생성하고, 제거하고, 검사합니다.

bool hash_init (struct hash *hash, hash_hash_func *hash_func,
                    hash_less_func *less_func, void *aux)

hash를 해시 테이블로 초기화하며, hash func는 해시 함수로, less func는 비교 함수로, aux는 보조 데이터(auxiliary data)로 사용합니다. 성공하면 true를 반환하고 실패하면 false를 반환합니다. hash_init()은 malloc()을 호출하며, 메모리를 할당할 수 없으면 실패합니다. 대부분 널 포인터(null pointer)인 aux에 대한 설명은 해시 보조 데이터(Hash Auxiliary Data)를 참조하십시오.

void hash_clear (struct hash *hash, hash_action_func *action)

hash에서 모든 원소를 제거합니다. hash는 이전에 hash_init()으로 초기화되어 있어야 합니다. action이 널이 아니면 해시 테이블의 각 원소에 대해 action을 한 번씩 호출하므로, 호출자는 원소가 사용하는 메모리나 다른 리소스를 해제할 기회를 얻습니다. 예를 들어 해시 테이블 원소가 malloc()으로 동적 할당되었다면, action은 그 원소를 free()할 수 있습니다. action을 호출한 뒤에는 hash_clear()가 해당 해시 원소의 메모리에 접근하지 않으므로 이는 안전합니다. 그러나 action은 hash_insert()나 hash_delete()처럼 해시 테이블을 수정할 수 있는 함수를 호출해서는 안 됩니다.

void hash_destroy (struct hash *hash, hash_action_func *action);

action이 널이 아니면 hash_clear() 호출과 같은 의미로 hash의 각 원소에 대해 action을 호출합니다. 그런 다음 hash가 보유한 메모리를 해제합니다. 이후 중간에 hash_init()을 다시 호출하지 않았다면, hash를 어떤 해시 테이블 함수에도 넘겨서는 안 됩니다.

size_t hash_size (struct hash *hash);

현재 hash에 저장된 원소 수를 반환합니다.

bool hash_empty (struct hash *hash);

현재 hash에 원소가 하나도 없으면 true를 반환하고, 하나 이상 있으면 false를 반환합니다.

검색 함수(Search Functions)

이 함수들은 각각 제공된 원소와 같다고 비교되는 원소를 해시 테이블에서 찾습니다. 검색 성공 여부에 따라 새 원소를 해시 테이블에 삽입하는 등의 동작을 수행하거나, 단순히 검색 결과를 반환합니다.

struct hash_elem *hash_insert (struct hash *hash, struct hash elem *element);

hash에서 element와 같은 원소를 찾습니다. 찾지 못하면 element를 hash에 삽입하고 널 포인터를 반환합니다. 테이블에 이미 element와 같은 원소가 있으면, hash를 수정하지 않고 그 원소를 반환합니다.

struct hash_elem *hash_replace (struct hash *hash, struct hash elem *element);

element를 hash에 삽입합니다. hash 안에 이미 element와 같은 원소가 있으면 그 원소를 제거합니다. 제거된 원소를 반환하며, hash에 element와 같은 원소가 없었다면 널 포인터를 반환합니다.

호출자는 반환된 원소와 관련된 리소스를 적절히 해제할 책임이 있습니다. 예를 들어 해시 테이블 원소가 malloc()으로 동적 할당되었다면, 더 이상 필요하지 않을 때 호출자가 그 원소를 free()해야 합니다.

다음 함수들에 전달되는 element는 해싱과 비교 목적으로만 사용됩니다. 실제로 해시 테이블에 삽입되지는 않습니다. 따라서 element 안에서는 키 데이터만 초기화하면 되고, 다른 데이터는 사용되지 않습니다. 원소 타입의 인스턴스를 지역 변수로 선언하고 키 데이터를 초기화한 뒤, 그 원소의 struct hash_elem 주소를 hash_find()나 hash_delete()에 넘기는 방식이 흔히 적절합니다. 예제는 해시 테이블 예제(Hash Table Example)를 참조하십시오. (큰 구조체는 지역 변수로 할당해서는 안 됩니다. 자세한 내용은 struct thread를 참조하십시오.)

struct hash_elem *hash_find (struct hash *hash, struct hash elem *element);

hash에서 element와 같은 원소를 찾습니다. 찾은 원소가 있으면 그 원소를 반환하고, 없으면 널 포인터를 반환합니다.

struct hash_elem *hash_delete (struct hash *hash, struct hash elem *element);

hash에서 element와 같은 원소를 찾습니다. 찾으면 hash에서 제거한 뒤 반환합니다. 그렇지 않으면 널 포인터를 반환하고 hash는 변경되지 않습니다.

호출자는 반환된 원소와 관련된 리소스를 적절히 해제할 책임이 있습니다. 예를 들어 해시 테이블 원소가 malloc()으로 동적 할당되었다면, 더 이상 필요하지 않을 때 호출자가 그 원소를 free()해야 합니다.

순회 함수(Iteration Functions)

이 함수들은 해시 테이블의 원소들을 순회할 수 있게 해 줍니다. 두 가지 인터페이스가 제공됩니다. 첫 번째 인터페이스에서는 각 원소에 작용할 액션 함수(action function)인 hash_action_func를 작성해 제공해야 합니다.

void hash_apply (struct hash *hash, hash action func *action);

hash의 각 원소에 대해 임의의 순서로 action을 한 번씩 호출합니다. action은 hash_insert()나 hash_delete()처럼 해시 테이블을 수정할 수 있는 함수를 호출해서는 안 됩니다. action은 원소 안의 키 데이터를 수정해서는 안 되지만, 그 밖의 데이터는 수정할 수 있습니다.

두 번째 인터페이스는 "반복자(iterator)" 데이터 타입을 기반으로 합니다. 관례적으로 반복자는 다음과 같이 사용합니다.

struct hash_iterator i;
hash_first (&i, h);
while (hash_next (&i)) {
    struct foo *f = hash_entry (hash_cur (&i), struct foo, elem);
    . . . do something with f . . .
}

struct hash_iterator;

해시 테이블 안의 한 위치를 나타냅니다. hash_insert()나 hash_delete()처럼 해시 테이블을 수정할 수 있는 함수를 호출하면, 해당 해시 테이블 안의 모든 반복자가 무효화됩니다.

struct hash와 struct hash_elem처럼 struct hash_elem도 불투명합니다.

void hash_first (struct hash iterator *iterator, struct hash *hash);

iterator를 hash의 첫 번째 원소 바로 앞 위치로 초기화합니다.

struct hash_elem *hash_next (struct hash iterator *iterator);

iterator를 hash의 다음 원소로 전진시키고 그 원소를 반환합니다. 남은 원소가 없으면 널 포인터를 반환합니다. hash_next()가 iterator에 대해 null을 반환한 뒤 다시 호출하면 정의되지 않은 동작(undefined behavior)이 발생합니다.

struct hash_elem *hash_cur (struct hash iterator *iterator);

iterator에 대해 hash_next()가 가장 최근에 반환한 값을 반환합니다. iterator에 대해 hash_first()가 호출된 뒤, hash_next()가 처음 호출되기 전에는 정의되지 않은 동작을 일으킵니다.

해시 테이블 예제(Hash Table Example)

해시 테이블에 넣고 싶은 struct page라는 구조체가 있다고 가정합시다. 먼저 struct page가 struct hash_elem 멤버를 포함하도록 정의합니다.

struct page {
    struct hash_elem hash_elem; /* Hash table element. */
    void *addr; /* Virtual address. */
    /* . . . other members. . . */
};

addr를 키로 사용하는 해시 함수와 비교 함수를 작성합니다. 포인터는 그 바이트 값을 기준으로 해시할 수 있고, 포인터 비교에는 < 연산자를 사용해도 충분합니다.

/* Returns a hash value for page p. */
unsigned
page_hash (const struct hash_elem *p_, void *aux UNUSED) {
  const struct page *p = hash_entry (p_, struct page, hash_elem);
  return hash_bytes (&p->addr, sizeof p->addr);
}

/* Returns true if page a precedes page b. */
bool
page_less (const struct hash_elem *a_,
           const struct hash_elem *b_, void *aux UNUSED) {
  const struct page *a = hash_entry (a_, struct page, hash_elem);
  const struct page *b = hash_entry (b_, struct page, hash_elem);

  return a->addr < b->addr;
}

(이 함수들의 프로토타입에서 UNUSED를 사용하면 aux가 사용되지 않는다는 경고가 억제됩니다. UNUSED에 대한 정보는 함수 및 매개변수 속성(Function and Parameter Attributes)을 참조하십시오. aux에 대한 설명은 해시 보조 데이터(Hash Auxiliary Data)를 참조하십시오.)

그러면 다음과 같이 해시 테이블을 만들 수 있습니다.

struct hash pages;
hash_init (&pages, page_hash, page_less, NULL);

이제 생성한 해시 테이블을 조작할 수 있습니다. p가 struct page에 대한 포인터라면, 다음으로 해시 테이블에 삽입할 수 있습니다.

hash_insert (&pages, &p->hash_elem);

pages에 같은 addr를 가진 페이지가 이미 들어 있을 가능성이 있다면 hash_insert()'의 반환값을 확인해야 합니다.

해시 테이블에서 원소를 검색하려면 hash_find()를 사용합니다. hash_find()는 비교 대상 원소를 인자로 받으므로 약간의 준비가 필요합니다. pages가 파일 스코프에 정의되어 있다고 가정할 때, 다음 함수는 가상 주소(virtual address)를 기준으로 페이지를 찾아 반환합니다.

/* Returns the page containing the given virtual address, or a null pointer if no such page exists. */
struct page *
page_lookup (const void *address) {
  struct page p;
  struct hash_elem *e;

  p.addr = address;
  e = hash_find (&pages, &p.hash_elem);
  return e != NULL ? hash_entry (e, struct page, hash_elem) : NULL;
}

여기서는 struct page가 충분히 작다고 가정하고 지역 변수로 할당했습니다. 큰 구조체는 지역 변수로 할당해서는 안 됩니다.

비슷한 함수는 hash_delete()를 사용해 주소로 페이지를 삭제할 수도 있습니다.

보조 데이터(Auxiliary Data)

위 예제처럼 단순한 경우에는 aux 매개변수가 필요하지 않습니다. 이런 경우에는 aux에 널 포인터를 hash_init()으로 전달하고, 해시 함수와 비교 함수에 전달되는 값은 무시하면 됩니다. (aux 매개변수를 사용하지 않으면 컴파일러 경고가 발생하지만, 예제처럼 UNUSED 매크로로 그 경고를 끌 수도 있고 그냥 무시할 수도 있습니다.)

aux는 해시 테이블 안 데이터의 어떤 속성이 상수이고 해싱이나 비교에 필요하지만, 데이터 항목 자체에는 저장되어 있지 않을 때 유용합니다. 예를 들어 해시 테이블의 항목이 고정 길이 문자열이지만 항목 자체가 그 고정 길이를 나타내지 않는다면, 그 길이를 aux 매개변수로 전달할 수 있습니다.

동기화(Synchronization)

해시 테이블은 내부 동기화를 수행하지 않습니다. 해시 테이블 함수 호출을 동기화하는 것은 호출자의 책임입니다. 일반적으로 hash_find()나 hash_next()처럼 해시 테이블을 검사하지만 수정하지 않는 함수는 여러 개가 동시에 실행될 수 있습니다. 그러나 이러한 함수들은 hash_insert()나 hash_delete()처럼 주어진 해시 테이블을 수정할 수 있는 함수와 동시에 안전하게 실행될 수 없으며, 주어진 해시 테이블을 수정할 수 있는 함수가 둘 이상 동시에 안전하게 실행될 수도 없습니다.

해시 테이블 원소 안의 데이터 접근을 동기화하는 것도 호출자의 책임입니다. 이 데이터에 대한 접근을 어떻게 동기화할지는 다른 자료구조와 마찬가지로 데이터가 설계되고 구성된 방식에 따라 달라집니다.
