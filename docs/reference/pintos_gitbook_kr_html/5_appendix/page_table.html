페이지 테이블(Page Table)

페이지 테이블(Page Table)

threads/mmu.c의 코드는 x86_64 하드웨어 페이지 테이블(page table)에 대한 추상 인터페이스입니다. Pintos의 페이지 테이블(프로젝트에서 사용하게 될 것)은 Intel 프로세서 문서에서 Page-Map-Level-4를 뜻하는 pml4라고 부릅니다. 이 테이블이 4단계(level)로 구성되기 때문입니다. 페이지 테이블 인터페이스는 내부 구조에 접근하기 편하도록 페이지 테이블을 나타내는 데 uint64_t *를 사용합니다. 아래 절에서는 페이지 테이블 인터페이스와 내부 구조를 설명합니다.

생성, 소멸, 활성화(Creation, Destruction, and Activation)

이 함수들은 페이지 테이블(page table)을 생성하고, 소멸시키고, 활성화합니다. 기본 Pintos 코드는 필요한 곳에서 이미 이 함수들을 호출하므로, 직접 호출할 필요는 없을 것입니다.

uint64_t * pml4_create (void);

새 페이지 테이블(page table)을 생성해 반환합니다. 새 페이지 테이블에는 Pintos의 일반적인 커널 가상 페이지 매핑(kernel virtual page mappings)은 포함되지만 사용자 가상 매핑(user virtual mappings)은 없습니다. 메모리를 얻을 수 없으면 null 포인터를 반환합니다.

void pml4_destroy (uint64_t *pml4);

pml4 자체와 pml4가 매핑하는 프레임(frame)을 포함하여 pml4가 보유한 모든 리소스를 해제합니다. 페이지 테이블(page table)의 모든 레벨에서 모든 리소스를 해제하기 위해 재귀적으로(recursively) pdpe_destroy, pgdir_destory, pt_destroy를 호출합니다. 

void pml4_activate (uint64 t *pml4)

pml4를 활성화합니다. 활성 페이지 테이블(active page table)은 CPU가 메모리 참조(memory reference)를 변환하는 데 사용하는 페이지 테이블입니다.

검사와 갱신(Inspection and Updates)

이 함수들은 페이지 테이블(page table)로 캡슐화된 페이지(page)에서 프레임(frame)으로의 매핑을 검사하거나 갱신합니다. 실행 중인 프로세스와 일시 중단된 프로세스의 페이지 테이블, 즉 활성 및 비활성 페이지 테이블 모두에서 동작하며, 필요에 따라 TLB를 플러시(flush)합니다.

bool pml4_set_page (uint64_t *pml4, void *upage, void *kpage, bool rw);

pd에 사용자 페이지(user page) upage에서 커널 가상 주소(kernel virtual address) kpage로 식별되는 프레임(frame)으로의 매핑을 추가합니다. rw가 true이면 페이지는 읽기/쓰기(read/write)로 매핑되고, 그렇지 않으면 읽기 전용(read-only)으로 매핑됩니다. 사용자 페이지 upage는 pml4에 이미 매핑되어 있으면 안 됩니다. 커널 페이지(kernel page) kpage는 palloc_get_page(PAL_USER)로 사용자 풀(user pool)에서 얻은 커널 가상 주소여야 합니다. 성공하면 true를, 실패하면 false를 반환합니다. 페이지 테이블(page table)에 필요한 추가 메모리를 얻을 수 없으면 실패합니다.

void * pml4_get_page (uint64_t *pml4, const void *uaddr);

pml4에서 uaddr에 매핑된 프레임(frame)을 찾습니다. uaddr이 매핑되어 있으면 그 프레임의 커널 가상 주소(kernel virtual address)를 반환하고, 매핑되어 있지 않으면 null 포인터를 반환합니다.

void pml4_clear_page (uint64_t *pml4, void *upage);

pml4에서 페이지를 “존재하지 않음(not present)”으로 표시합니다. 이후 그 페이지에 접근하면 폴트(fault)가 발생합니다. 해당 페이지에 대한 페이지 테이블(page table)의 다른 비트들은 보존되므로 접근 비트(accessed bit)와 더티 비트(dirty bit)(다음 절 참조)를 확인할 수 있습니다. 해당 페이지가 매핑되어 있지 않으면 이 함수는 아무 효과가 없습니다.

접근 비트와 더티 비트(Accessed and Dirty Bits)

x86_64 하드웨어는 각 페이지의 페이지 테이블 엔트리(page table entry, PTE)에 있는 한 쌍의 비트를 통해 페이지 교체 알고리즘(page replacement algorithms) 구현을 일부 지원합니다. 페이지를 읽거나 쓸 때마다 CPU는 해당 페이지의 PTE에서 접근 비트(accessed bit)를 1로 설정하고, 어떤 쓰기든 발생하면 CPU는 더티 비트(dirty bit)를 1로 설정합니다. CPU는 이 비트들을 0으로 재설정하지 않지만 OS는 그렇게 할 수 있습니다. 이 비트들을 올바르게 해석하려면 별칭(aliases)을 이해해야 합니다. 별칭은 같은 프레임(frame)을 가리키는 두 개 이상의 페이지를 뜻합니다. 별칭이 있는 프레임에 접근하면 접근 비트와 더티 비트는 접근에 사용된 페이지의 페이지 테이블 엔트리 하나에서만 갱신됩니다. 다른 별칭들의 접근 비트와 더티 비트는 갱신되지 않습니다.

bool pml4_is_dirty (uint64_t *pml4, const void *vpage);
bool pml4_is_accessed (uint64_t *pml4, const void *vpage);

pml4가 vpage에 대해 더티 비트(dirty bit) 또는 접근 비트(accessed bit)가 표시된 페이지 테이블 엔트리(page table entry)를 포함하면 true를 반환합니다. 그렇지 않으면 false를 반환합니다.

void pml4_set_dirty (uint64_t *pml4, const void *vpage, bool dirty);
void pml4_set_accessed (uint64_t *pml4, const void *vpage, bool accessed);

pml4에 page에 대한 페이지 테이블 엔트리(page table entry)가 있으면, 그 더티 비트(dirty bit)(또는 접근 비트(accessed bit))를 주어진 값으로 설정합니다.
