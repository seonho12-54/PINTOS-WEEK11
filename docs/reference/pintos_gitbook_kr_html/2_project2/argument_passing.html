인자 전달(Argument Passing)

인자 전달(Argument Passing)

process_exec()에서 사용자 프로그램(user program)의 인자를 설정하십시오.

x86-64 호출 규약(Calling Convention)

이 절은 64비트 x86-64 Unix 구현에서 일반 함수 호출에 사용하는 호출 규약의 중요한 부분을 요약합니다. 간결성을 위해 일부 세부 사항은 생략했습니다. 더 자세한 내용은 System V AMD64 ABI를 참고하십시오.

호출 규약은 다음과 같이 동작합니다.

  1. 사용자 수준 애플리케이션은 정수 인자를 전달할 때 순서대로 %rdi, %rsi, %rdx, %rcx, %r8, %r9 레지스터를 사용합니다.
  2. 호출자(caller)는 다음 명령어의 주소, 즉 반환 주소(return address)를 스택(stack)에 푸시하고 피호출자(callee)의 첫 번째 명령어로 점프합니다. x86-64 명령어 하나인 CALL이 이 두 작업을 모두 수행합니다.
  3. 피호출자(callee)가 실행됩니다.
  4. 피호출자에 반환값이 있으면 그 값을 RAX 레지스터에 저장합니다.
  5. 피호출자는 스택에서 반환 주소를 꺼내고, x86-64 RET 명령어를 사용해 그 주소가 가리키는 위치로 점프하여 반환합니다.

세 개의 int 인자를 받는 함수 f()를 생각해 봅시다. 다음 그림은 f()가 f(1, 2, 3)으로 호출되었다고 가정할 때, 위의 3단계가 시작되는 시점에 피호출자에서 보이는 예시 스택 프레임(stack frame)과 레지스터 상태를 보여 줍니다. 초기 스택 주소는 임의의 값입니다.

                             +----------------+
stack pointer --> 0x4747fe70 | return address |
                             +----------------+
RDI: 0x0000000000000001 | RSI: 0x0000000000000002 | RDX: 0x0000000000000003

프로그램 시작 세부 사항(Program Startup Details)

Pintos의 사용자 프로그램용 C 라이브러리는 _start()를 사용자 프로그램의 진입점(entry point)으로 지정하며, 이는 lib/user/entry.c에 있습니다. 이 함수는 main()을 감싸는 래퍼(wrapper)이며, exit()는 main()이 반환할 때 호출됩니다.

void
_start (int argc, char *argv[]) {
    exit (main (argc, argv));
}

커널(kernel)은 사용자 프로그램(user program)의 실행을 시작시키기 전에 초기 함수의 인자들을 레지스터에 넣어야 합니다. 인자는 일반 호출 규약과 같은 방식으로 전달됩니다.

다음 예시 명령의 인자를 어떻게 처리할지 생각해 봅시다. /bin/ls -l foo bar.

  1. 명령을 단어로 나눕니다. /bin/ls, -l, foo, bar.

  2. 단어들을 스택(stack)의 맨 위에 놓습니다. 이 단어들은 포인터를 통해 참조되므로 순서는 중요하지 않습니다.

  3. 각 문자열의 주소와 널 포인터 센티널(null pointer sentinel)을 오른쪽에서 왼쪽 순서로 스택에 푸시합니다. 이것들이 argv의 원소입니다. 널 포인터 센티널은 C 표준이 요구하는 대로 argv[argc]가 널 포인터가 되게 합니다. 이 순서는 argv[0]이 가장 낮은 가상 주소(virtual address)에 놓이도록 보장합니다. 단어 정렬(word-aligned) 접근은 정렬되지 않은 접근보다 빠르므로, 최상의 성능을 위해 첫 번째 푸시 전에 스택 포인터(stack pointer)를 8의 배수로 내림 정렬하십시오.

  4. %rsi가 argv, 즉 argv[0]의 주소를 가리키게 하고 %rdi를 argc로 설정합니다.

  5. 마지막으로 가짜 "반환 주소(return address)"를 푸시합니다. 진입 함수는 결코 반환하지 않지만, 그 스택 프레임(stack frame)은 다른 함수와 같은 구조를 가져야 합니다.

아래 표는 사용자 프로그램(user program)이 시작되기 직전의 스택(stack) 상태와 관련 레지스터를 보여 줍니다. 스택은 아래쪽으로 자란다는 점에 유의하십시오.

주소(Address) 이름(Name) 데이터(Data) 타입(Type)
0x4747fffc argv[3][...] 'bar\0' char[4]
0x4747fff8 argv[2][...] 'foo\0' char[4]
0x4747fff5 argv[1][...] '-l\0' char[3]
0x4747ffed argv[0][...] '/bin/ls\0' char[8]
0x4747ffe8 word-align 0 uint8_t[]
0x4747ffe0 argv[4] 0 char *
0x4747ffd8 argv[3] 0x4747fffc char *
0x4747ffd0 argv[2] 0x4747fff8 char *
0x4747ffc8 argv[1] 0x4747fff5 char *
0x4747ffc0 argv[0] 0x4747ffed char *
0x4747ffb8 반환 주소(return address) 0 void (*) ()

RDI: 4 | RSI: 0x4747ffc0

이 예시에서 스택 포인터는 0x4747ffb8로 초기화됩니다. 위에 보인 것처럼 여러분의 코드는 USER_STACK에서 스택을 시작해야 합니다. 이는 include/threads/vaddr.h에 정의되어 있습니다.

비표준 hex_dump() 함수는 <stdio.h>에 선언되어 있으며, 여러분의 인자 전달(argument passing) 코드를 디버깅(debugging)할 때 유용할 수 있습니다.

인자 전달(argument passing)을 구현하십시오.

현재 process_exec()는 새 프로세스(process)에 인자를 전달하는 기능을 지원하지 않습니다. 이 기능을 구현하려면 process_exec()를 확장하여, 인자로 단순히 프로그램 파일 이름만 받는 대신 공백을 기준으로 단어를 나누게 하십시오. 첫 번째 단어는 프로그램 이름, 두 번째 단어는 첫 번째 인자이며 그 뒤도 같은 방식입니다. 즉, process_exec("grep foo bar")는 grep을 실행하면서 foo와 bar라는 두 인자를 전달해야 합니다.

명령줄(command line) 안에서 여러 개의 공백은 하나의 공백과 같으므로, process_exec("grep foo bar")는 원래 예시와 동일합니다. 명령줄 인자의 길이에 합리적인 제한을 둘 수 있습니다. 예를 들어 인자가 한 페이지(4 kB)에 들어가는 경우로 제한할 수 있습니다. pintos 유틸리티가 커널(kernel)에 전달할 수 있는 명령줄 인자에는 이와 별개로 128바이트 제한이 있습니다.

인자 문자열은 원하는 방식으로 파싱(parse)해도 됩니다. 막막하다면 strtok_r()를 살펴보십시오. 이 함수는 include/lib/string.h에 프로토타입이 있고 lib/string.c에 자세한 주석과 함께 구현되어 있습니다. 프롬프트에서 man strtok_r를 실행하여 man 페이지를 확인할 수도 있습니다.
