실행 파일 쓰기 거부(Deny Write on Executables)

실행 파일(executable)에 대한 쓰기를 거부하십시오.

실행 파일로 사용 중인 파일에 쓰기를 거부하는 코드를 추가하십시오. 많은 운영체제(OS)가 이렇게 하는 이유는, 프로세스(process)가 디스크에서 변경 중인 코드를 실행하려 할 때 예측할 수 없는 결과가 생길 수 있기 때문입니다. 이 기능은 프로젝트 3에서 가상 메모리(virtual memory)를 구현한 뒤 특히 중요하지만, 지금 구현해도 해가 되지 않습니다.

열려 있는 파일에 대한 쓰기를 막으려면 file_deny_write()를 사용할 수 있습니다. 그 파일에 file_allow_write()를 호출하면 쓰기가 다시 허용됩니다. 단, 다른 열기 주체가 그 파일에 대한 쓰기를 거부하고 있지 않은 경우에만 그렇습니다. 파일을 닫아도 쓰기는 다시 허용됩니다. 따라서 프로세스(process)의 실행 파일에 대한 쓰기를 거부하려면, 그 프로세스(process)가 실행 중인 동안 해당 파일을 계속 열어 두어야 합니다.
