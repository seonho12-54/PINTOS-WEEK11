알람 시계(Alarm Clock)

timer_sleep()을 다시 구현하십시오. 이 함수는 devices/timer.c에 정의되어 있습니다.

현재 동작하는 구현이 제공되어 있지만, 이 구현은 바쁜 대기(busy wait)를 수행합니다. 즉, 충분한 시간이 지날 때까지 루프를 돌며 현재 시간을 확인하고 thread_yield()를 호출합니다. 바쁜 대기를 피하도록 다시 구현하십시오.

void timer_sleep (int64_t ticks);

호출한 스레드(thread)의 실행을 시간이 적어도 x 타이머 틱(timer tick)만큼 진행될 때까지 중지합니다. 시스템이 유휴(idle) 상태가 아닌 한, 해당 스레드가 정확히 x틱 후에 깨어날 필요는 없습니다. 알맞은 시간만큼 기다린 뒤 준비 큐(ready queue)에 넣기만 하면 됩니다.

timer_sleep()은 커서를 초당 한 번 깜박이는 경우처럼 실시간으로 동작하는 스레드에 유용합니다. timer_sleep()의 인자는 밀리초나 다른 단위가 아니라 타이머 틱으로 표현됩니다. 초당 TIMER_FREQ개의 타이머 틱이 있으며, TIMER_FREQ는 devices/timer.h에 정의된 매크로입니다. 기본값은 100입니다. 이 값을 변경하면 많은 테스트가 실패할 가능성이 크므로 변경하지 않는 것을 권장합니다.

밀리초, 마이크로초, 나노초 단위로 특정 시간 동안 잠들기 위한 별도 함수인 timer_msleep(), timer_usleep(), timer_nsleep()가 존재하지만, 필요할 때 이 함수들은 자동으로 timer_sleep()을 호출합니다. 이 함수들을 수정할 필요는 없습니다. 알람 시계(alarm clock) 구현은 이후 프로젝트에는 필요하지 않지만, 프로젝트 4에서는 유용할 수 있습니다.
