스왑 인/아웃(swap in/out)

스왑 인/아웃(swap in/out)

메모리 스와핑(memory swapping)은 물리 메모리 사용률을 극대화하기 위한 메모리 회수 기법입니다. 메인 메모리의 프레임(frame)이 모두 할당되면 시스템은 사용자 프로그램의 추가 메모리 할당(memory allocation) 요청을 처리할 수 없습니다. 한 가지 해결책은 현재 사용되지 않는 메모리 프레임을 디스크로 스왑 아웃(swap out)하는 것입니다. 이렇게 하면 일부 메모리 자원이 해제되어 다른 애플리케이션이 사용할 수 있게 됩니다.

스와핑(swapping)은 운영체제(operating system)가 수행합니다. 시스템이 메모리가 부족하다는 것을 감지했지만 메모리 할당(memory allocation) 요청을 받으면, 스왑 디스크(swap disk)로 축출(evict)할 페이지를 선택합니다. 그런 다음 메모리 프레임(frame)의 정확한 상태가 디스크에 복사됩니다. 프로세스(process)가 스왑 아웃(swap out)된 페이지에 접근하려고 하면 OS는 정확한 내용을 다시 메모리로 가져와 페이지를 복구합니다.

축출 대상으로 선택된 페이지는 익명 페이지(anonymous page)이거나 파일 기반 페이지(file-backed page)일 수 있습니다. 이 절에서는 각 경우를 처리합니다.

모든 스와핑 작업은 명시적으로 호출되지 않고 함수 포인터(function pointer)로 호출됩니다. 이 함수 포인터들은 struct page_operations file_ops의 멤버이며, 각 페이지의 초기화 함수(initializer)에 대한 작업(operation)으로 등록됩니다.

익명 페이지(anonymous page)

vm_anon_init과 anon_initializer를 vm/anon.c에서 수정합니다. 익명 페이지(anonymous page)에는 이를 위한 백업 저장소(backing storage)가 없습니다. 익명 페이지의 스와핑을 지원하기 위해 스왑 디스크(swap disk)라는 임시 백업 저장소를 제공합니다. 익명 페이지의 스왑(swap)을 구현하는 데 이 스왑 디스크를 사용합니다.

void vm_anon_init (void);

이 함수에서 스왑 디스크(swap disk)를 설정해야 합니다. 또한 스왑 디스크에서 비어 있는 영역과 사용 중인 영역을 관리할 자료구조가 필요합니다. 스왑 영역도 PGSIZE (4096 bytes) 단위로 관리됩니다.

bool anon_initializer (struct page *page, enum vm_type type, void *kva);

이 함수는 익명 페이지(anonymous page)의 초기화 함수(initializer)입니다. 스와핑을 지원하려면 anon_page에 일부 정보를 추가해야 합니다.

이제 익명 페이지의 스와핑을 지원하도록 anon_swap_in과 anon_swap_out을 vm/anon.c에서 구현합니다. 페이지가 스왑 인(swap in)되려면 먼저 스왑 아웃(swap out)되어야 하므로, anon_swap_out을 구현한 뒤 anon_swap_in을 구현하는 것이 좋습니다. 데이터 내용을 스왑 디스크(swap disk)로 옮기고, 안전하게 다시 메모리로 가져와야 합니다.

static bool anon_swap_in (struct page *page, void *kva);

스왑 디스크(swap disk)에서 데이터 내용을 읽어 메모리로 가져와 익명 페이지(anonymous page)를 스왑 인(swap in)합니다. 데이터의 위치는 스왑 디스크 안에 있으며, 페이지가 스왑 아웃(swap out)될 때 page 구조체에 저장되어 있어야 합니다. 스왑 테이블(swap table)을 업데이트하는 것을 잊지 마십시오(스왑 테이블 관리(Managing the Swap Table) 참조).

static bool anon_swap_out (struct page *page);

메모리의 내용을 디스크로 복사하여 익명 페이지(anonymous page)를 스왑 디스크(swap disk)로 스왑 아웃(swap out)합니다. 먼저 스왑 테이블(swap table)을 사용해 디스크에서 비어 있는 스왑 슬롯(swap slot)을 찾은 다음, 데이터 페이지를 해당 슬롯에 복사합니다. 데이터의 위치는 page 구조체에 저장되어야 합니다. 디스크에 더 이상 비어 있는 슬롯이 없으면 커널 패닉(kernel panic)을 발생시킬 수 있습니다.

파일 매핑 페이지(file-mapped page)

파일 기반 페이지(file-backed page)의 내용은 파일에서 오므로, mmap된 파일을 백업 저장소(backing store)로 사용해야 합니다. 즉 파일 기반 페이지를 축출하면 해당 페이지가 매핑되었던 파일에 다시 기록됩니다. file_backed_swap_in, file_backed_swap_out을 vm/file.c에서 구현합니다. 설계에 따라 file_backed_init과 file_initializer를 수정할 수도 있습니다.

static bool file_backed_swap_in (struct page *page, void *kva);

파일에서 내용을 읽어 kva에 있는 페이지를 스왑 인(swap in)합니다. 파일 시스템(file system)과 동기화해야 합니다.

static bool file_backed_swap_out (struct page *page);

내용을 파일에 다시 기록하여 페이지를 스왑 아웃(swap out)합니다. 먼저 페이지가 더티(dirty)인지 확인하는 것이 좋습니다. 더티가 아니면 파일의 내용을 수정할 필요가 없습니다. 페이지를 스왑 아웃한 후에는 페이지의 더티 비트(dirty bit)를 끄는 것을 잊지 마십시오.
