쓰기 시 복사(copy-on-write) (Extra)

Pintos에서 쓰기 시 복사(copy-on-write) 메커니즘을 구현하십시오.

쓰기 시 복사(copy-on-write)는 동일한 물리 페이지(physical page) 인스턴스를 사용하여 복제 작업을 더 빠르게 수행할 수 있게 하는 자원 관리 기법입니다. 어떤 자원을 여러 프로세스(process)가 사용하고 있다면, 충돌이 발생하지 않도록 보통 각 프로세스는 해당 자원의 자체 복사본을 가져야 합니다. 그러나 그 자원이 수정되지 않고 읽히기만 한다면 물리 메모리(physical memory)에 여러 복사본을 둘 필요가 없습니다.

예를 들어, fork를 통해 새 프로세스(process)가 생성되었다고 가정해 보겠습니다. 자식은 부모의 자원을 상속해야 하므로 데이터를 자신의 가상 주소 공간(virtual address space)에 복제해야 합니다. 일반적으로 가상 메모리(virtual memory)에 내용을 추가하려면 물리 페이지(physical page)를 할당하고, 데이터를 프레임(frame)에 쓰고, 페이지 테이블(page table)에 가상->물리 매핑(virtual->physical mapping)을 추가해야 합니다. 이 단계들은 상당한 시간이 걸릴 수 있습니다.

하지만 쓰기 시 복사(copy-on-write) 기법을 사용하면 자원의 새 복사본을 위해 새 물리 페이지(physical page)를 할당하지 않습니다. 기술적으로는 그 내용이 이미 물리 메모리(physical memory)에 존재하기 때문입니다. 따라서 가상 주소가 이제 자식의 메모리 공간에 있는 상태로, 자식 프로세스(process)의 페이지 테이블(page table)에 가상->물리 매핑(virtual->physical mapping)만 추가합니다. 그러면 부모와 자식은 같은 물리 페이지(physical page)에 있는 동일한 데이터에 접근합니다. 그렇지만 이들은 여전히 별도의 가상 주소 공간(virtual address space)을 통해 격리되어 있으며, 같은 프레임(frame)을 참조하고 있다는 사실은 OS만 알고 있습니다. 공유 자원의 내용을 프로세스(process) 중 하나가 수정하려고 할 때에만, 그 프로세스는 자신을 위해 새 물리 페이지(physical page)에 별도의 복사본을 만듭니다. 따라서 실제 복사 작업은 첫 번째 쓰기까지 지연됩니다.

즉, OS는 쓰기 시 복사(copy-on-write) 페이지에 대한 쓰기 시도를 감지할 수 있어야 합니다. 이를 위해 OS는 "쓰기 보호(write-protect)" 메커니즘을 사용합니다. 아이디어는 간단합니다. 쓰기 접근에서 페이지 폴트(page fault)를 발생시키는 것입니다. 메모리 관리 시스템의 지원을 받아 쓰기 보호된(write-protected) 페이지를 쓰기 불가능한 것으로 표시하기만 하면 쉽게 구현할 수 있습니다.

여러분은 fork에 대해서만 쓰기 시 복사(copy-on-write)를 구현하면 됩니다. 자식 프로세스(process)가 부모 프로세스(process)로부터 자원을 상속할 때, 그 자원은 자식이 수정하려고 시도하기 전까지 동일한 물리 데이터를 참조할 수 있습니다. 쓰기 보호된 모든 페이지는 축출(eviction) 후보입니다.

쓰기 시 복사(copy-on-write)에 대해서는 기본 테스트 케이스만 제공합니다. 가능한 모든 경우를 고려해야 합니다(작은 힌트: 파일 기반 페이지(file-backed page)의 공유도 구현해야 합니다). 이 추가 프로젝트의 채점은 숨겨진 테스트 케이스로도 수행됩니다.
