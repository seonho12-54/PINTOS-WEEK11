채점

채점

과제는 테스트 결과를 기준으로 채점합니다.

테스트

테스트 결과 점수는 우리가 제공하는 테스트를 기준으로 합니다. 각 프로젝트에는 여러 테스트가 있으며, 각 테스트의 이름은 tests로 시작합니다. 제출물을 완전히 테스트하려면 프로젝트의 build 디렉터리에서 make check를 실행하십시오. 그러면 각 테스트를 빌드하고 실행한 뒤, 테스트마다 pass 또는 fail 메시지를 출력합니다. 테스트가 실패하면 make check는 실패 원인에 대한 세부 정보도 일부 출력합니다. 모든 테스트를 실행한 뒤에는 make check가 테스트 결과 요약도 출력합니다. 개별 테스트를 하나씩 실행할 수도 있습니다. 주어진 테스트 t는 출력을 t.output에 기록하고, 그런 다음 스크립트가 출력을 pass 또는 fail로 채점하여 판정을 t.result에 기록합니다. 단일 테스트를 실행하고 채점하려면 .result 파일을 build 디렉터리에서 명시적으로 만드십시오. 예를 들면 make tests/threads/alarm-multiple.result입니다. make가 테스트 결과가 최신이라고 말하지만 그래도 다시 실행하고 싶다면, make clean을 실행하거나 .output 파일을 직접 삭제하십시오. 기본적으로 각 테스트는 실행 중이 아니라 완료 시점에만 피드백을 제공합니다. 원한다면 make check VERBOSE=1처럼 make 명령줄에 VERBOSE=1을 지정하여 각 테스트의 진행 상황을 볼 수 있습니다. 모든 테스트와 관련 파일은 pintos/src/tests에 있습니다. 제출물을 테스트하기 전에, 올바른 테스트가 사용되도록 그 디렉터리의 내용을 수정되지 않은 깨끗한 원본으로 교체합니다. 따라서 디버깅에 도움이 된다면 일부 테스트를 수정할 수 있지만, 우리는 원본 테스트를 실행합니다. 모든 소프트웨어에는 버그가 있으므로, 우리의 테스트 중 일부에도 결함이 있을 수 있습니다. 테스트 실패가 여러분 코드의 버그가 아니라 테스트의 버그라고 생각한다면 알려 주십시오. 검토한 뒤 필요하면 수정하겠습니다. 테스트 스위트를 공개하는 우리의 관대함을 악용하려고 하지 마십시오. 여러분의 코드는 우리가 제공하는 테스트 케이스에서만이 아니라 일반적인 경우에도 제대로 동작해야 합니다. 예를 들어 실행 중인 테스트 케이스의 이름을 기준으로 커널의 동작을 명시적으로 바꾸는 것은 허용되지 않습니다. 이런 식으로 테스트 케이스를 우회하려는 시도에는 점수를 주지 않습니다. 여러분의 해법이 이와 관련해 애매한 영역에 있을 수 있다고 생각한다면 우리에게 문의하십시오.

제출

아카이브 파일을 만드는 명령을 제공합니다. threads, userprog, vm, filesys 디렉터리가 있는 pintos 프로젝트 루트 디렉터리로 이동하십시오. TEAM=YOUR_TEAM_NUMBER make archive를 입력합니다(예: TEAM=5 make archive). 그러면 같은 디렉터리에 team5.tar.gz가 생깁니다. 이 아카이브 파일을 제출하십시오. 중복 제출은 하지 마십시오. 팀당 한 번 제출하면 충분합니다.
